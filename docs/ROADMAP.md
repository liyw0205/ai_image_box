## 0.3.2 任务页操作

状态：初版已完成。已补齐任务页取消、失败/取消/成功任务重试、已完成任务清理和中文状态展示。

目标：让队列页可以直接处理常见生成任务流转。

- 排队中和运行中任务可取消。
- 失败、取消和成功任务可重新提交为新任务。
- 可清理当前队列里的已完成任务，不影响历史记录。
- 多任务执行时创作页提交状态跟随队列空闲状态。

## 0.3.3 任务恢复与详情

状态：初版已完成。已补齐 APP 重启后未完成任务恢复、不可恢复任务失败落库、provider 失败响应预览写入 attempts、任务详情和历史详情弹窗。

目标：提升真机调试和失败复现效率。

- 启动时恢复 queued/running 任务到队列。
- 渠道缺失或禁用时将未完成任务标记为失败并写入历史。
- 失败 attempt 保存 HTTP 状态、耗时、错误和脱敏响应预览。
- 任务页和历史页可查看详情。

## 0.3.4 Provider 错误中文化

状态：初版已完成。已补齐 OpenAI-compatible 渠道测试和文生图失败的常见 HTTP、网络、TLS、Base URL、响应解析和图片下载错误中文提示。

目标：让真机调试时能快速判断是密钥、地址、额度、模型、网络还是响应格式问题。

- HTTP 400/401/403/404/413/422/429/5xx 输出中文原因和原始错误摘要。
- DNS、连接失败、超时、TLS 失败输出中文原因。
- Base URL、提示词为空、响应无图片、下载/解码失败输出中文原因。
- 失败信息继续进入任务 attempt 和详情弹窗。

## 0.3.5 结果预览与渠道体验

状态：初版已完成。已补齐创作页生成结果图片预览、保存到公共 Pictures/AI Image Box、历史结果公共导出、请求/响应详情分段、provider 类型预设选择、模型名类型推断和 `extra.model_types` 保存。

目标：减少渠道配置成本，并让生成结果可以直接查看和导出。

- 新增渠道时 provider 类型可手输，也可点预设。
- 拉取模型后根据模型名推断 OpenAI/Gemini/Grok/Agnes/视频接口类型，未匹配时默认 OpenAI 图像格式。
- 生成成功后创作页显示首张图片，不再只显示私有路径。
- 历史和创作页可将私有目录结果保存到公共图片目录。
- 详情按请求信息、请求数据、响应信息、响应数据、响应结果分段。

## 0.3.6 模型类型编辑与按钮适配

状态：初版已完成。模型列表中每个模型的推断类型现在可以手动修改，保存后写入 `extra.model_types` 并在创作页按模型类型展示/提交；多图结果增加横向缩略图切换；任务、历史、渠道和配置弹窗按钮改为两列换行布局，避免文字被遮挡或省略。

目标：解决真机上模型类型不可改和按钮文案显示不全的问题。

- 模型选择弹窗每行增加“改类型”，可在 OpenAI 图像、Gemini、Agnes、Grok、视频格式之间选择，也可恢复自动推断。
- 创作目标读取模型级 `extra.model_types`，提交时保留模型对应 provider 类型。
- 多张生成结果在创作页显示首图和缩略图列表，可点缩略图切换预览。
- 多按钮区域不再一行硬挤，改为两列多行和自适应高度。

## 0.3.7 渠道编辑体验整理

状态：初版已完成。渠道编辑顶部改为图片/视频两种模式，视频预设先支持 Grok 和 Seedance；扩展 JSON 增加可视化/JSON 切换，既能点选预设写入 JSON，也能直接编辑 JSON；模型列表行改为纵向排版，长模型名不再被按钮遮挡。

目标：继续降低渠道配置成本，并修复真机小屏上的文字遮挡。

- 渠道编辑不再暴露底层 provider 类型输入框，先以图片/视频模式组织配置。
- 视频可视化预设写入 `extra.video_provider`，当前包含 `grok` 和 `seedance`。
- 扩展 JSON 默认显示可视化配置，切换 JSON 后可手动编辑原始 JSON。
- 创作页渠道信息展示图片/视频模式，不再展示 provider 类型。

## 0.3.8 默认模型移除与模型类型可视化

状态：初版已完成。渠道编辑移除了默认模型输入；渠道类型顶部图片/视频切换增加当前状态反馈；扩展配置的可视化页改为修改已配置模型的类型，不再显示错误的视频预设按钮；模型类型保留 Grok 视频和 Seedance 视频选项。

目标：让渠道配置逻辑和上游模型列表一致，避免无效字段和误导按钮。

- 渠道编辑不再展示默认模型，保存时默认使用已启用模型列表的第一项作为内部兼容值。
- 图片/视频切换明确显示当前渠道类型，点击后会更新隐藏的 provider 类型。
- 可视化扩展配置列出已配置模型，可逐个修改模型类型并同步到 `extra.model_types`。
- JSON 模式仍可直接编辑扩展 JSON，切回可视化会刷新显示。

## 0.3.9 图生图入口和模型选择调整

状态：初版已完成。渠道不再区分图片/视频类型；创作页目标切换从顺序点击改为选择弹窗；创作页加入可选参考图，选择后按图生图提交；OpenAI-compatible 图像适配器初步支持 `/v1/images/edits` multipart 请求。

目标：把“渠道是接口，模型决定能力”的配置方式落到创作流程。

- 渠道编辑只保留通用接口配置，图片/视频能力通过模型级 `extra.model_types` 修改。
- 创作页显示可选参考图预览，任务参数和 attempts 会记录 `reference_images`。
- 生成管理器根据是否存在参考图自动切换 `TEXT_TO_IMAGE` / `IMAGE_TO_IMAGE`。
- OpenAI-compatible 图生图初版使用首张参考图作为 `image` 表单字段。

## 0.3.10 多图像 provider 初接入

状态：初版已完成。模型类型 `gemini_image`、`agnes_image`、`grok_image` 已接入 provider registry，创作页按模型级类型路由到对应适配器；Gemini 和 Agnes 支持参考图，Grok 先支持文生图。

目标：让“模型类型”不只停留在配置层，而是能实际决定请求格式。

- Gemini 使用 `generateContent` JSON 请求，参考图写入 `inline_data`。
- Agnes 使用 OpenAI-like `/v1/images/generations`，参考图以 data URL 写入 `extra_body.image`。
- Grok 使用 `/v1/images/generations`，写入 `aspect_ratio`、`resolution` 和 `response_format`。
- `ProviderRegistry.targetsFor` 会读取 `extra.model_types`，为后续自动回退和模型优先级打基础。

## 0.4.5 跨渠道自动回退

状态：初版已完成。生成执行器会按“当前选择模型 -> 当前渠道启用模型 -> 所有启用渠道模型”的顺序展开目标，失败后继续尝试下一个可用 provider/model，并在 attempts、任务详情和历史记录里保留真实渠道、provider 类型和模型。

- `ProviderRegistryGenerationExecutor` 接入 `ChannelStore`，运行时读取所有启用渠道作为全局回退池。
- Provider 调用不再复用原始渠道参数，跨渠道尝试会使用对应渠道的 Base URL、API Key、超时和代理。
- 成功结果和历史记录会写入最终成功的渠道、provider 类型和模型。
- attempts JSON 增加 `channel_id`，失败和成功详情都能区分跨渠道尝试来源。

## 0.4.6 渠道配置增强

- 扩展配置可视化页增加“批量类型”和“使用推断”，已配置模型不再必须逐个修改。

## 0.5.0 生视频抽象

状态：最小闭环已完成。已接入 `GenericAsyncVideoAdapter`，模型类型为 `openai_compatible_video`、`grok_video`、`seedance_video` 或 `generic_async_video` 时，生成执行器会按参考图有无自动切换文生视频/图生视频；adapter 支持提交、解析同步视频 URL、解析 job id、轮询、下载视频到私有 `generated_videos`，并记录 job id、poll URL、HTTP 状态、响应预览和 attempts。轮询由应用级队列执行，前台服务通知、远端 job 恢复和视频缩略图均已完成。

目标：支持异步生视频。

- `VideoProviderAdapter`：用 `GenericAsyncVideoAdapter` 先承载通用异步视频。
- submit/poll/download 通用流程：已完成。
- `ProviderJob` 持久化：job id、poll URL 进入结果 metadata 和 attempt response。
- 轮询前台服务：已完成，运行期间显示低打扰通知，队列空闲后自动停止。
- 取消本地轮询：可取消当前协程任务，远端取消接口未接入。
- 视频结果保存已完成；缩略图在 0.5.1 完成。

验收：

- 文生视频/图生视频任务可从 Running 进入 provider 轮询流程。
- APP 重启后可从持久化 job id 和 poll URL 精确恢复远端任务并继续轮询。
- 成功后下载视频，失败后保留 job id、poll URL、响应预览和错误。

## 0.5.2 远端视频任务恢复

状态：初版已完成。异步视频提交取得 job id 后会立即把 job id、poll URL、result URL 和响应预览写入任务参数；应用重启恢复运行中视频任务时直接继续轮询原远端 job，不再重复提交生成请求。

- provider request 支持 `resumeJob` 和 job 更新回调。
- 首次提交及每次轮询解析到的新 job 状态都会持久化。
- 重启恢复时从 `provider_job` 还原远端任务并继续 poll/download。
- 渠道缺失或 job URL 无效时继续按现有失败记录流程落库。

## 0.5.3 应用级后台队列

状态：初版已完成。GenerationManager、ChannelStore 和 GenerationStore 已提升到 Application 生命周期，Activity 销毁或屏幕旋转不再关闭运行中的生成队列；重新进入界面后继续观察同一队列和任务持久化状态。系统级 ForegroundService 通知已在 0.5.4 完成。

- GenerationManager 由 `AIImageBoxApp` 单例持有。
- Activity 不再在 `onDestroy` 取消运行中任务。
- provider job 持久化回调移动到应用级执行器。
- 页面重建复用相同 ChannelStore、GenerationStore 和队列。

## 0.5.4 前台服务通知

- AndroidManifest 注册 `GenerationForegroundService`。
- Android 8.0+ 创建低打扰生成任务通知渠道。
- 通知点击可返回主界面，运行期间保持 ongoing 状态。
- 队列进入空闲状态后自动 `stopForeground` 和 `stopSelf`。

## 0.6.0 历史与素材库

目标：让结果可管理、可复用。

- 历史筛选：模式、成功/失败、provider、模型和提示词搜索已完成；时间范围已完成。
- 复用参数再次生成：单条记录、当前筛选结果首条和显式多选批量复用均已完成；批量操作会预检查渠道、模型和参考素材。
- 以结果作为参考图/视频：图片和视频结果均可在历史中作为素材选择并预览；当前内置 provider 只接受图片参考，视频素材会在提交前阻止，待有真实上游视频引用契约后再开放。
- 批量清理缓存：已完成，保留配置和历史索引。
- 素材详情：已完成，显示类型、尺寸、时长、大小、存在状态、路径和元数据。

验收：
