# AI Image Box 路线图

## 0.1.0 工程骨架

状态：已完成。`bash pack_ai_image_box.sh` 已成功生成 `../AIImageBox_0.1.0_debug.apk`。

目标：先跑起来一个可安装 APP。

- Android Gradle 工程。
- `com.aiimagebox` 包名。
- MainActivity + 四 Tab。
- 深色主题和基础布局。
- 应用私有目录初始化。
- Termux 打包脚本。

验收：

- `bash pack_ai_image_box.sh` 生成 debug APK。
- 冷启动显示“创作”工作台。
- 竖屏底栏、横屏导航轨可切换。

## 0.2.0 渠道管理

状态：基础渠道管理已完成。已实现 `ProviderChannel`、`ModelTarget`、`channels.json`、Android Keystore API Key 加密保存、渠道新增、编辑、复制、启用、禁用、删除、渠道页列表、provider 模板、Base URL 基础校验，以及上游模型拉取、筛选和保存到启用模型。

目标：本地可配置 provider。

- ProviderChannel 数据模型。
- 本地 `channels.json`。
- API Key 加密保存。
- 渠道列表、新增、编辑、复制、禁用、删除。
- 测试连接和模型列表。
- 配置导出时脱敏。

验收：

- 添加 OpenAI-compatible 渠道。
- 保存后重启仍存在。
- 删除渠道不影响历史文件。

## 0.3.0 文生图

状态：初版已完成。已接入 `StudioForm` 创作表单、`GenerationManager` 队列、`GenerationStore` 任务/历史持久化、OpenAI-compatible `/v1/images/generations` 文生图、结果图片落盘、任务页队列状态、历史页最近记录和渠道模型列表拉取/筛选/保存。

目标：完成第一条端到端生成链路。

- OpenAI-compatible image adapter。
- 标准 `GenerationRequest` / `GenerationResult`。
- `GenerationManager` 队列。
- 创作页提交文生图。
- 结果下载到本地缓存。
- 失败记录和错误展示。

验收：

- 文生图成功显示图片。
- HTTP 失败显示响应预览。
- 超时可恢复，任务可重试。

## 0.3.1 文生图回归增强

状态：初版已完成。已补齐多图结果完整落盘、图片尺寸记录、历史缩略图和历史记录参数复用入口。

目标：让 0.3.0 的端到端链路更接近可用初版。

- 保存 provider 返回的全部图片，而不是只保留第一张。
- 历史页显示首张图片缩略图。
- 历史记录可一键复用提示词、渠道、模型、比例、分辨率和数量。
- 成功状态显示本次保存的结果数量和路径预览。

## 0.3.2 任务页操作

状态：初版已完成。已补齐任务页取消、失败/取消/成功任务重试、已完成任务清理和中文状态展示。

目标：让队列页可以直接处理常见生成任务流转。

- 排队中和运行中任务可取消。
- 失败、取消和成功任务可重新提交为新任务。
- 可清理当前队列里的已完成任务，不影响历史记录。
- 多任务执行时创作页提交状态跟随队列空闲状态。

## 0.3.3 任务恢复与详情

状态：初版已完成。已补齐 APP 重启后未完成任务恢复、不可恢复任务失败落库、provider 失败响应预览写入 attempts、任务详情和历史详情弹窗。

目标：提升真机调试和失败复现效率。

- 启动时恢复 queued/running 任务到队列。
- 渠道缺失或禁用时将未完成任务标记为失败并写入历史。
- 失败 attempt 保存 HTTP 状态、耗时、错误和脱敏响应预览。
- 任务页和历史页可查看详情。

## 0.3.4 Provider 错误中文化

状态：初版已完成。已补齐 OpenAI-compatible 渠道测试和文生图失败的常见 HTTP、网络、TLS、Base URL、响应解析和图片下载错误中文提示。

目标：让真机调试时能快速判断是密钥、地址、额度、模型、网络还是响应格式问题。

- HTTP 400/401/403/404/413/422/429/5xx 输出中文原因和原始错误摘要。
- DNS、连接失败、超时、TLS 失败输出中文原因。
- Base URL、提示词为空、响应无图片、下载/解码失败输出中文原因。
- 失败信息继续进入任务 attempt 和详情弹窗。

## 0.4.0 图生图和多 provider 回退

目标：迁移插件的主要生图能力。

- 图生图参考图。
- Gemini image adapter。
- Agnes image adapter。
- Grok image adapter。
- 模型优先级。
- 多 provider 尝试记录。

验收：

- 参考图参与请求。
- 第一个 provider 失败后自动尝试下一个。
- 历史记录里能看到 attempts。

## 0.5.0 生视频抽象

目标：支持异步生视频。

- `VideoProviderAdapter`。
- submit/poll/download 通用流程。
- `ProviderJob` 持久化。
- 轮询前台服务。
- 取消本地轮询。
- 视频结果保存和缩略图。

验收：

- 文生视频任务可从 Running 进入 Polling。
- APP 重启后可恢复轮询。
- 成功后下载视频，失败后保留 job id 和错误。

## 0.6.0 历史与素材库

目标：让结果可管理、可复用。

- 历史筛选：时间、模式、provider、模型、成功/失败。
- 复用参数再次生成。
- 以结果作为参考图/视频。
- 批量清理缓存。
- 素材详情。

验收：

- 1000 条记录列表不卡顿。
- 清理缓存不删除配置。
- 记录里的本地文件缺失时有明确状态。

## 0.7.0 子代理流水线

目标：支持无限扩展的生成前后处理。

- `AgentRegistry`。
- `AgentPipeline` 配置。
- 提示词增强子代理。
- 参考图分析子代理。
- 可选审核子代理。
- 元数据写入子代理。

验收：

- 可启用/禁用/排序子代理。
- 子代理不写死数量上限。
- 每个子代理耗时和输出进入任务记录。

## 0.8.0 诊断、导入导出与稳定化

目标：可维护、可迁移、可定位问题。

- 诊断包。
- 配置导出导入。
- 请求/响应脱敏。
- 缓存占用统计。
- 横屏 D-pad 回归。
- 常见 provider 错误中文化。

验收：

- 诊断包不含 API Key。
- 导入配置后渠道可用，缺 key 有明确提示。
- 横屏键盘和遥控可完成基础流程。
