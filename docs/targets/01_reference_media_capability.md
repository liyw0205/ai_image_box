# 目标 01：参考素材能力校验

状态：已完成（0.8.7）

## 背景

创作页已经允许图片和视频作为参考素材，但不同 provider 的输入字段并不相同。当前通用图生图和图生视频请求使用图片字段，直接把视频字节作为参考输入会形成无效请求。

## 范围

- 在提交前识别参考素材 MIME 类型。
- 图片模型只接受图片参考素材。
- 当前图生视频模板只接受图片素材；视频素材保留在历史和预览中，但不作为不受支持的二进制请求内容发送。
- 给用户明确、可本地化的错误状态，不创建无效队列任务。
- 保留可扩展能力模型，后续 provider 声明支持视频参考时再开放。

## 验收

- 选择图片素材后，图片和视频模型可正常创建任务。
- 选择视频素材后，当前内置模型不会创建无效任务，并显示明确原因。
- 任务参数、历史复用与现有图片参考流程不回归。
- 增加纯 JVM 单元测试覆盖素材类型判定。
- `testDebugUnitTest` 和 `bash pack_ai_image_box.sh` 通过。
