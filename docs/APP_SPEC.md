# AI Image Box APP 产品规格

## 一句话目标

做一个 Android 原生多接口生图、生视频工作台：用户可以配置多个模型渠道，提交文生图、图生图、文生视频、图生视频任务，查看任务进度、历史记录、生成素材和接口诊断。

## 产品模式

- 不设置客户端 token 额度、日限额或调用次数限制。
- 不设置固定子代理数量上限；子代理采用注册表模式，能按配置继续扩展。
- 不把审核做成强制硬限制；审核、OCR、提示词增强、结果复核都作为可配置子代理。
- 只保留工程必要的保护：请求超时、并发上限、缓存上限、单文件大小上限、取消任务和错误隔离。

这里的“无限制”是产品策略：APP 不做业务额度限制。实际请求仍受设备内存、网络、provider 自身限制和用户配置影响。

## 目标用户流程

1. 首次打开
   - 进入工作台，不做营销页。
   - 可直接添加接口渠道，或进入离线空状态。
   - API Key 和渠道配置只保存在本机。

2. 添加渠道
   - 填渠道名、base URL、API Key、启用模型、超时、代理和扩展 JSON。
   - 渠道本身不区分图片/视频；每个模型可在扩展配置里单独设置图片或视频接口类型。
   - 支持测试连接、拉取模型列表、保存、复制、禁用、删除。

3. 发起生成
   - 在“创作”页选择模式：文生图、图生图、文生视频、图生视频。
   - 输入提示词，或从预设中选择手办化、真人化、变 COS、Q 版化等常用图生图提示词。
   - 可添加参考图、比例、分辨率、时长、风格、种子、数量等参数。
   - 选择模型优先级或使用自动回退。
   - 提交后进入任务队列，实时显示当前 provider、阶段、耗时和错误。

4. 查看结果
   - 生成完成后展示图片或视频缩略图。
   - 支持保存到本地、分享、复制参数、再次生成、以结果作为参考图/参考视频继续生成。
   - 失败任务保留请求摘要、响应摘要、错误、重试按钮。

5. 管理素材和历史
   - 历史按时间、类型、provider、模型、成功/失败筛选。
   - 素材库保留请求图、生成图、生成视频、任务 JSON 和响应摘要。
   - 可清理缓存、导出诊断包、导出/导入配置。

## 信息架构

底部导航或横屏导航轨：

| Tab | 作用 |
| --- | --- |
| 创作 | 发起文生图、图生图、文生视频、图生视频 |
| 任务 | 查看队列、运行中任务、失败重试、批量取消 |
| 历史 | 查看生成记录、筛选、复用参数、结果管理 |
| 渠道 | 管理 provider、模型、优先级、测试和诊断 |

右侧抽屉或顶栏入口：

- 设置：并发、超时、缓存、默认比例、默认分辨率、日志级别。
- 子代理：提示词增强、参考图分析、OCR、审核、provider 选择、结果复核、元数据写入。
- 数据：缓存占用、记录占用、清理、导出诊断包。

## 核心对象

| 对象 | 说明 |
| --- | --- |
| ProviderChannel | 一个接口渠道，包含 provider 类型、base URL、API Key、模型列表和扩展参数 |
| ModelTarget | 一次可调用目标，由渠道加模型组成 |
| GenerationTask | 用户提交的一次任务，可包含图或视频请求 |
| GenerationRequest | 标准化请求参数，屏蔽不同 provider 的字段差异 |
| GenerationResult | 标准化结果，包含本地文件、远程 URL、元数据和错误 |
| AgentStep | 可插拔子代理步骤，如提示词增强、OCR、审核、结果复核 |
| GenerationRecord | 任务记录，用于历史、诊断、复用参数 |

## 任务状态

```text
Draft -> Queued -> Running -> Polling -> Downloading -> Succeeded
                                |          |             |
                                v          v             v
                              Failed     Canceled      Failed
```

状态要求：

- 每个状态变化写入任务记录。
- 异步视频接口必须保存 provider job id 和轮询参数。
- 失败要保留可复现信息：provider、model、request id、HTTP 状态、响应预览、耗时。
- 取消任务必须停止本地轮询和下载；无法取消远端任务时记录说明。

## 首版范围

首版只做本地单机 APP：

- Android 手机和平板优先，横屏遥控兼容参考 MediaVault。
- Provider 配置、任务、历史和文件都保存在应用私有目录。
- 支持 OpenAI 风格生图接口、Gemini 生图接口，以及至少一种异步生视频接口抽象。
- 支持手动添加任意 OpenAI-compatible image/video endpoint。
- 暂不做账号、多设备同步、云端队列、Web 管理端和服务器端中转。

## 当前实现状态

0.8.11 已完成生图与生视频链路、AgentRegistry/AgentPipeline、四类内置代理、远端 job 恢复、ForegroundService 通知以及历史素材库核心功能：渠道管理、API Key 加密、模型拉取/筛选/类型编辑、渠道模板、快速校验、文生图、图生图参考图、OpenAI-compatible/Gemini/Agnes/Grok 图像 adapter、通用异步视频 submit/poll/download、视频结果落盘和公共 Movies 导出、同渠道和跨渠道回退、任务取消/重试/清理、重启恢复、历史缩略图、参数复用、多图落盘和公共目录导出。

Grok 与 Seedance 已提供独立渠道模板和请求字段映射；仍需使用有效账户执行真实接口回归。横屏已使用 NavigationRail，并补充初始焦点、左右焦点出口、系统焦点高亮和聚焦控件自动滚入可视区域；完整遥控真机回归仍需设备验收。配置与诊断文件已可通过系统文件创建器保存到用户选择的位置。历史筛选结果可直接复用首条记录，图片和视频结果均可在历史中预览和选择；当前内置 provider 仅接受图片参考素材，视频引用会在提交前被明确阻止，避免产生无效任务。视频模型显示 1 至 30 秒时长控件并完整进入任务与历史。历史页支持显式选择筛选记录、预检查渠道/模型/参考素材并确认后批量创建新任务。异步视频链路已有本地 MockWebServer 契约回归，覆盖 submit、poll、下载与失败脱敏。视频渠道在保存和校验前会验证 provider、提交路径和轮询占位符。GitHub Actions 现在运行单元测试、构建 APK 并校验 APK 包名和版本元数据，真机回归使用不含凭据的记录模板。诊断包会对任务、历史、嵌入 JSON 和代理元数据执行递归脱敏。

## 非目标

- 不做纯 WebView 套壳。
- 不把 MediaVault 的视频播放、刮削、远程 SMB/FTP/WebDAV 媒体库逻辑搬进来。
- 不把 AstrBot 插件的 Python 运行时直接嵌入 Android；只迁移 provider 设计和数据结构。
- 不在客户端实现业务额度系统。
